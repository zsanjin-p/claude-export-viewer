# Claude 对话导出查看器

一个**单文件、完全离线**的工具，用来浏览、搜索、导出 Claude.ai 导出的对话数据（`conversations.json`）。

- 不需要安装任何东西，不需要服务器
- 双击 `claude对话导出查看器.html` 即可用浏览器打开使用
- 所有解析、搜索都在你本机浏览器里完成，**不会上传任何数据**，断网也能用
- 支持超大文件（几十 MB 的导出也能秒开），得益于用 Web Worker 后台解析

---

## 快速开始

1. 双击打开 `claude对话导出查看器.html`
2. 根据你的导出数据情况，选择下面两种模式之一（见下文）
3. 导入完成后，左侧是对话列表，点击任意一条即可查看详情

再次打开这个 HTML 文件时，会自动恢复上次导入的数据（缓存在浏览器本地），不用重新导入。

---

## 两种导入模式

### 模式一：单个文件

适合只有一份 `conversations.json` 的情况。

- 点击或拖拽 `conversations.json` 进入拖放区
- 如果同时选中 `users.json`（按住 Ctrl / Cmd 多选），会自动识别账户信息（姓名、邮箱），显示在对话详情里

### 模式二：导出文件夹（批量 / 多账户）

适合 Claude 官方"导出账户数据"下载下来的完整压缩包解压后的样子，结构通常是这样：

```
导出的对话/
├─ data-08c8b6fd-...batch-0000/
│   ├─ conversations.json
│   └─ users.json
├─ data-402462f3-...batch-0000/
│   ├─ conversations.json
│   └─ users.json
└─ data-c13ce19e-...batch-0000/
    ├─ conversations.json
    └─ users.json
```

使用方法：

1. 切换到「导出文件夹」标签
2. 点击「选择导出文件夹」，选中最外层的根目录（比如上面例子里的 `导出的对话`）
3. 工具会自动递归扫描所有子文件夹，把每个 `data-xxx-batch-0000` 里配套的 `conversations.json` + `users.json` 找出来并合并导入

**关于"刷新"功能**：如果你的浏览器支持（Chrome / Edge 等基于 Chromium 的浏览器），工具会记住这个文件夹的访问权限。之后如果文件夹里又新增了 `data-xxx-batch-000x` 批次（比如又导出了一次新数据放进同一个目录），直接点左上角的「刷新」按钮，就会自动重新扫描并把新增的对话合并进来，不用重新选择文件夹。

如果浏览器不支持这个能力（比如 Firefox、Safari），会自动切换成兼容模式：可以正常一次性选择整个文件夹导入，但无法"刷新识别新增"，每次都需要重新选择文件夹。

## 多账户切换

如果导入的数据里包含多个账户（多个 `users.json` 里的用户），左下角会出现一个用户切换器，显示当前选中的账户头像、姓名、邮箱。

点击可以：
- 切换到「全部用户」，查看所有账户的对话
- 切换到某个具体账户，只看这个账户名下的对话（对话列表、统计数字、全文搜索范围都会跟着筛选）

如果导出数据里只有一个账户，或者没有提供 `users.json`，这个切换器不会显示，直接看全部对话即可。

## 功能一览

- **对话列表**：按最近更新 / 创建时间 / 消息数 / 标题排序，带每条对话的简介预览
- **关键词过滤**：在搜索框输入即时过滤标题和摘要
- **全文搜索**：输入关键词后按 Enter，会在所有对话的完整内容（包括工具调用、工具结果、附件内容）里搜索，按命中次数排序
- **对话详情**：还原用户 / Claude 消息气泡，包含：
  - 正文文本（支持代码块高亮显示）
  - 💭 思考过程（可折叠）
  - 🔧 工具调用 / ↩ 工具结果（可折叠，避免长对话把界面撑爆，右上角有「展开全部」按钮）
  - 📎 附件内容预览
- **导出 Markdown**：查看某个对话时，右上角「导出为 Markdown」可以把整个对话保存成 `.md` 文件
- **本地缓存**：解析结果存在浏览器 IndexedDB 里，重新打开工具会自动恢复；加载界面左下角可以「清除本地缓存」

---

## 隐私说明

这个工具不包含任何网络请求代码。文件读取、JSON 解析、全文搜索、Markdown 导出全部在你当前打开的浏览器标签页里完成。数据只会保存在你本机浏览器的 IndexedDB 中（用于下次快速恢复），不会发送到任何服务器。你可以随时点击「清除本地缓存」删除这些数据，或者直接清空浏览器数据。

## 常见问题

**Q: 换了一个新的 conversations.json，想重新导入？**
点击左上角的「+」（重新加载）按钮，会回到导入界面，重新选择文件或文件夹即可，会完全替换掉之前导入的数据。

**Q: 文件夹模式点了"刷新"没反应 / 提示没有权限？**
大概率是浏览器不支持 File System Access API（比如用的是 Firefox），或者是文件夹的访问授权过期了。前一种情况请用 Chrome / Edge；后一种情况重新点一次「选择导出文件夹」重新授权即可。

**Q: 支持的浏览器？**
建议使用最新版 Chrome、Edge，功能最完整（含文件夹刷新）。Firefox、Safari 可以正常使用单文件模式和文件夹一次性导入，但不支持刷新识别新增批次。

**Q: 文件很大，会不会卡死？**
不会。JSON 解析放在 Web Worker 后台线程里进行，不会阻塞界面；实测几十 MB 的导出文件解析通常在 1 秒以内完成。

**Q: 能不能在没有网络的电脑上用？**
可以。这是一个纯本地单文件工具，没有任何外部依赖（不加载 CDN、不发请求），复制到任何一台装了现代浏览器的电脑上，双击打开即可使用。
